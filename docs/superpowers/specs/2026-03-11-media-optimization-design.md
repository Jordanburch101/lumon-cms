# Media Optimization Design

Upload-time media optimization for the Lumon Payload template. Protects the pipeline so raw uploads never reach storage unprocessed, regardless of what a client uploads.

## Goals

- Optimize images on upload (resize, convert to WebP, strip metadata)
- Optimize videos on upload via ffmpeg (transcode, cap resolution, optional audio strip)
- Generate blur placeholders (base64) for instant loading UX
- Show image size variants in the Payload admin panel
- ffmpeg is optional — graceful fallback when unavailable
- No original files retained — optimized replaces raw

## Architecture

```
Upload flow:
  Image → Payload native formatOptions/resizeOptions → optimized WebP → S3
        → afterChange hook → generate blurDataURL → update document
  Video → afterChange hook → queue job → ffmpeg (async) → optimized MP4 + blurDataURL → S3
```

Images are optimized natively by Payload's built-in sharp integration via upload config (`formatOptions`, `resizeOptions`). Blur placeholders are generated in an `afterChange` hook. Videos process asynchronously via Payload's Jobs Queue (transcoding can take minutes).

## 1. Media Collection Config

**File**: `src/collections/Media.ts`

### Upload Settings

```ts
upload: {
  mimeTypes: ['image/*', 'video/*'],
  formatOptions: {
    format: 'webp',
    options: { quality: 82 },
  },
  resizeOptions: {
    width: 2560,
    height: 2560,
    fit: 'inside',
    withoutEnlargement: true,
  },
  // withMetadata defaults to false — EXIF/metadata is stripped automatically
  imageSizes: [
    {
      name: 'thumbnail',
      width: 400,
      height: 300,
      position: 'centre',
    },
    {
      name: 'card',
      width: 768,
      height: 512,
    },
    {
      name: 'hero',
      width: 1920,
      height: undefined, // no height constraint, preserves aspect ratio
      withoutEnlargement: true,
    },
  ],
  adminThumbnail: 'thumbnail',
  focalPoint: true,
  crop: true,
}
```

Notes:
- `formatOptions` converts all uploaded images to WebP at quality 82 before storage
- `resizeOptions` caps the source image at 2560px on the longest edge (no upscaling)
- `withMetadata` defaults to `false` in Payload — EXIF/metadata is stripped automatically
- `imageSizes` variants are generated from the already-optimized WebP source
- SVGs are not processed by sharp (Payload skips non-raster formats)
- Animated GIFs/WebP: converting to static WebP loses animation — this is acceptable for this template's use case (marketing imagery). If a fork needs animated support, override `formatOptions` per-collection.
- `card` uses 768x512 (3:2 landscape) to match the card layouts in the design

### New Fields

| Field | Type | Purpose |
|-------|------|---------|
| `alt` | text (required) | Accessibility text (existing) |
| `stripAudio` | checkbox, default `true` | Toggle audio stripping on videos. Shown only when mimeType is `video/*` via `admin.condition`. Toggling re-queues the video optimization job. |
| `blurDataURL` | text, `admin.hidden: true` | Base64-encoded 16px-wide blurred WebP. Generated automatically after upload. Used by `next/image` `placeholder="blur"`. Typically 200-400 bytes. |

## 2. Blur Placeholder Generation (afterChange Hook)

**File**: `src/collections/hooks/generateBlurDataURL.ts`

An `afterChange` hook on the Media collection. Runs after the file is stored by Payload.

### Trigger

- `operation === 'create'` with an image file
- `operation === 'update'` when a new file is uploaded
- Skip if `context.skipBlurGeneration` is set (loop protection)

### Pipeline (Images)

1. Check `doc.mimeType.startsWith('image/')` — skip if not an image
2. Read the stored file (from `req.file.data` buffer if available, or fetch from `doc.url`)
3. Sharp pipeline: resize to **16px wide** (preserve aspect ratio), blur (sigma 10), encode as WebP base64
4. Update the document with `blurDataURL`:

```ts
await req.payload.update({
  collection: 'media',
  id: doc.id,
  data: { blurDataURL: `data:image/webp;base64,${base64}` },
  context: { skipBlurGeneration: true },
  req,
})
```

The `context.skipBlurGeneration` flag prevents the `afterChange` hook from re-triggering on the update, avoiding an infinite loop.

### Pipeline (Videos)

Blur placeholder for videos is generated by the video optimization job (Section 3), not this hook.

## 3. Video Optimization (Async)

### Hook

**File**: `src/collections/hooks/optimizeVideo.ts`

An `afterChange` hook on the Media collection.

#### Trigger

- Upload is a video (`mimeType.startsWith('video/')`) on `create`
- `operation === 'update'` when a new video file is uploaded (`req.file` is present)
- `stripAudio` field is toggled on `update` (compare `doc.stripAudio` vs `previousDoc.stripAudio`)
- Skip if `context.skipVideoOptimization` is set (loop protection)

#### Behavior

1. Check ffmpeg availability via `isFFmpegAvailable()`
2. If unavailable: log warning, skip processing — video stores as-is
3. If available: queue `optimizeVideo` job via Payload Jobs Queue

### Job Task

**File**: `src/jobs/optimizeVideo.ts`

Registered in `payload.config.ts` under `jobs.tasks`.

#### Input Schema

| Field | Type | Description |
|-------|------|-------------|
| `mediaId` | number (required) | The Media document ID to process |

#### Processing Steps

1. Fetch the Media document to get current file URL and `stripAudio` value
2. Download the file from S3 to a temp directory
3. Run ffmpeg:
   - Transcode to **H.264/MP4** (`-c:v libx264`)
   - Cap at **1080p**: `-vf "scale=min(1920\,iw):min(1080\,ih):force_original_aspect_ratio=decrease,scale=trunc(iw/2)*2:trunc(ih/2)*2"` (second scale ensures even dimensions required by H.264)
   - Target bitrate **4-6 Mbps** (`-b:v 5M -maxrate 6M -bufsize 12M`)
   - If `stripAudio` is true: `-an` (no audio)
   - If `stripAudio` is false: `-c:a aac -b:a 128k`
   - Web-optimized: `-movflags +faststart`
4. Upload processed file back to S3, replacing the original
5. Generate blur placeholder: extract frame at 1s (`-ss 1 -vframes 1`), pipe through sharp (16px wide, blur, base64 WebP)
6. Update Media document with new `filesize`, `mimeType` (`video/mp4`), and `blurDataURL` — use `context: { skipVideoOptimization: true, skipBlurGeneration: true }` to prevent re-triggering hooks
7. Clean up temp files

#### Error Handling

- Retries: 2 (Payload Jobs Queue built-in)
- On failure: log error, leave original file intact

## 4. ffmpeg Utility

**File**: `src/lib/ffmpeg.ts`

Note: This lives in `src/lib/` (not `src/core/lib/`) because it's project-specific, not upstream-safe agnostic code.

### Exports

```ts
isFFmpegAvailable(): Promise<boolean>
```

Cached runtime check. Runs `ffmpeg -version` once, caches the result for the process lifetime.

```ts
runFFmpeg(args: string[]): Promise<{ stdout: string; stderr: string }>
```

Wrapper around `child_process.execFile` with:
- Configurable timeout (default 5 minutes)
- Promise-based API
- Stderr capture for error reporting

## 5. Config Changes

### `payload.config.ts`

Add jobs configuration:

```ts
jobs: {
  tasks: [optimizeVideoTask],
  autoRun: [
    { cron: "* * * * *" },
  ],
}
```

Import and register the `optimizeVideo` task from `src/jobs/optimizeVideo.ts`. The `autoRun` cron config runs the job queue every minute — without it, jobs would be queued but never executed.

### `next.config.ts`

Add S3 bucket hostname to `images.remotePatterns`:

```ts
images: {
  remotePatterns: [
    {
      protocol: 'https',
      hostname: 'images.unsplash.com',
    },
    {
      protocol: 'https',
      hostname: '*.railway.app',
    },
  ],
}
```

This allows `next/image` to optimize S3-hosted media. Without this, images served from the Railway S3 bucket will fail.

## 6. Frontend Integration

### Blur Placeholders

Components that render images update to use the `blurDataURL` from the Payload media object:

```tsx
<Image
  src={getMediaUrl(media)}
  placeholder={media.blurDataURL ? 'blur' : 'empty'}
  blurDataURL={media.blurDataURL}
  // ... other props
/>
```

A helper function extracts `blurDataURL` from the media object alongside the existing `getMediaUrl()`:

```ts
export function getBlurDataURL(
  media: string | { blurDataURL?: string } | undefined | null
): string | undefined {
  if (!media || typeof media === 'string') return undefined
  return media.blurDataURL || undefined
}
```

### No Other Frontend Changes

- Components already use `next/image` with responsive `sizes` attributes
- The `imageSizes` variants are available via the Payload media object (e.g., `media.sizes.thumbnail.url`)
- Video components already use `<video>` elements — the optimized MP4 is served from the same URL

## Dependencies

- **sharp** (v0.34.5) — already installed
- **ffmpeg** — optional system dependency, graceful fallback
- No new npm packages required
